Hear our story

Hacker Pile was founded by a college student looking for an easier way to push code between computers.

Why we do what we do

Our mission is to provide a quick, easy way for developers to move code between machines.

Hacker Pile was built to make things easier for developers who are just starting out. Learning the ins and outs of version control is both difficult and time consuming for beginners, and we think that there should be an easier alternative.

We care about using the right tool for the job. Version control is necessary for bigger projects, however for many beginner projects creating a remote repository is at the very least overkill.

Developers

Alex Pereira

As the founder of Hacker Pile, Alex had a vision of making learning to code much easier for those who are constantly on the go. He commuted to college, which meant constantly back and forth between his laptop and desktop at home.

As of version 1.0.0 (current release), Alex was the only developer who contributed to the project.
